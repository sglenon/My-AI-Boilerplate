---
name: code-critique
description: Explain selected code for a beginner, then critique its limitations, readability, engineering principles, and improvement opportunities.
---

Explain the selected code to me.
- Explain the code in terms of its purpose and functionality
- Explain the LOGIC and SYNTAX to me (as I am a noobie dev)
- Use simple language and avoid jargon
- Provide examples if possible
- Be concise and to the point

Then critique the code for me
- What are its limitations?
- What can be improved?
- Is it following best software engineering principles? (DRY, Type, SOLID, KISS)
- Is it self-readable even without verbose docstrings and comments? Or does it follow abstract naming
